# 🏭 신규 앱 출시 준비 표준 — 네이티브(Android) 버전

> 목적: 새 앱은 **만들 때부터 출시 준비 완료** 상태. (PWA 표준의 네이티브 대응판)
> A(자동) = 템플릿에 항상 포함 / B(수동) = 출시 때 사람이 스토어에서.

## A. 템플릿에 항상 포함 (공장 표준)
- Kotlin + Jetpack Compose(Material3), minSdk 24 / targetSdk 34
- 릴리즈 서명 설정(keystore.properties 패턴) — release 빌드 서명 가능
- `.gitignore` 에 키스토어·빌드산출물 제외 (키 절대 커밋 금지)
- **Ryu 배지 아이콘**: 앱 고유 아이콘 우상단에 흰 원+블루 R+쥐 (배지는 store_assets.py가 자동 합성)
- 밀도별 런처 PNG (mdpi~xxxhdpi), 콘솔 에러 0 빌드
- 스토어 자료 생성기(tools/store_assets.py): 아이콘·배너(1024x578)·스크린샷(731x1300)

## B. 출시할 때만 (수동)
1. applicationId·app_name 변경, (필요시) 아이콘 아트 교체
2. 업로드 키스토어 생성 → keystore.properties 또는 서명 마법사로 AAB 빌드
3. 원스토어/플레이 등록: 바이너리(AAB) 업로드, 스토어 자료, 연령등급, 가격
4. 개인정보 URL·이메일 = 공통 고정값 재사용
5. 검증요청 → 배포, `_출시관리.md` 대장에 한 줄 추가
