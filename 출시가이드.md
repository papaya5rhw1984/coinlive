# 출시 가이드 (네이티브 Android)

> PWA 방식(PWABuilder·GitHub Pages·assetlinks) 대신, Android Studio에서 **서명된 AAB를 직접 빌드해 스토어에 업로드**한다.

## 공통 고정값 (Ryu)
- 개인정보처리방침 URL: https://papaya5rhw1984.github.io/privacy/
- 문의 이메일: papaya5rhw@naver.com
- 패키지ID 규칙: `io.github.papaya5rhw1984.<앱이름>` (소문자, 출시 후 변경 불가)

## 1) 업로드 키스토어 (앱마다 1회)
Android Studio: Build ▸ Generate Signed App Bundle / APK ▸ Android App Bundle ▸ Create new...
- Key store path: `!!!앱출시하기\store_원본파일(지우면안돼_키있음)\<앱>\<앱>-release.jks`
- 비밀번호·alias 입력 후 생성. **.jks와 비밀번호는 분실 금지(잃으면 업데이트 영원히 불가).**

## 2) 서명된 AAB 빌드
Build ▸ Generate Signed App Bundle ▸ 기존 키스토어 선택 ▸ release ▸ Create
→ `app/release/app-release.aab`

## 3) 원스토어 등록
- 신규 바이너리: 유형 AAB + 서명키 "ONE store가 앱 서명키를 관리 보호" → AAB 업로드
- 판매정보: 아이콘(512)·그래픽배너(1024x578)·스크린샷(2~8장)·설명·키워드
- 연령등급: IARC 설문(앱마다 새로) → 보통 전체이용가
- 가격: 무료(권장 첫 출시) / 개인정보 URL·이메일 = 공통 고정값
- 검증요청 → 통과 시 배포
